CGIWrap - Setting up your Account

The following are instructions for setting up your directories and scripts for execution on UMR's test web server.

The instructions are intended for use with AFS. It is assumed that you are familiar with writing CGI scripts and writing HTML documents for the WWW. If not, see one of the URL's listed on the Web Page Design page.

Note: All scrips that are precompiled before use have to be compiled on a HP-UX machine (saucer/rocket). This is because the WWW servers are being run on HP-UX machines.

For this example, the script is "email.pl". The user id is "joeminer".

  • Create the directories needed to hold html documents and scripts
    1. cd
    2. mkdir public_html
    3. mkdir public_html/cgi-bin

  • Set minimal required permissions on directories
    1. fs setacl . system:anyuser l
    2. fs setacl public_html system:anyuser rl
    3. fs setacl public_html/cgi-bin system:anyuser rl

  • Install the script into the cgi-bin directory

  • Set the proper execution permissions on the file
    1. chmod u+rx email.pl

The script is executed using the following URL:
http://wwwcgi.umr.edu/cgi-bin/cgiwrap/joeminer/email.pl

If you wish to see debugging output for your CGI, specify cgiwrapd instead of cgiwrap.

When using cgiwrap with AFS based accounts, you need to keep in mind that scripts are run un-authenticated. This means that any files that are accessed in AFS directories are accessed as "system:anyuser". This is important when considering how your script will return data to you.

Currently, the easiest way to return data to the user is to email the results of the script. Obviously, this only applies to scripts that are supposed to be used for submitting data - scripts that simply return information to the web client are not affected by this.

CGIwrap can be used in forms, just use the above URL's as the ACTION of the form.
